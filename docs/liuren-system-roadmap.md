# 大六壬观事系统建设路线图

目标不是做一个简单的吉凶页面，而是做一套完整的观事、起课、断事、行动、复盘、学习系统。

## 第一阶段：问事收束

让用户先把问题问清楚，再起课。

- 增加判断重点：成败、时机、风险、对方态度、结果走向。
- 增加观察周期：当日、三日、一周、一月、自定义。
- 根据事项类型、对象、判断重点、观察周期自动生成更具体的问题。
- 在断语中明确：本次课到底在判断什么，不让结果泛泛而谈。

验收标准：
- 同一时间问不同事项，解读明显不同。
- 同一事项选择不同判断重点，行动建议和复盘点不同。
- 案例保存时记录判断重点和观察周期。

## 第二阶段：严肃排盘

把起课骨架做扎实。

- 节气月将，而不是简单按公历月份近似。
- 准确日干支和时辰处理。
- 昼夜贵人规则完善。
- 完整四课、三传、天盘、地盘、天将展示。
- 增加课体、空亡、刑冲合害、驿马、贵人等基础标签。

验收标准：
- 排盘结果可追溯。
- 页面同时提供白话版和术语版。

## 第三阶段：事项断课引擎

同一张课，按不同所问取不同类象。

- 投资求财：仓位、确认、虚动、风险。
- 合作人事：主客、信任、边界、关键人。
- 合同回款：承诺、流程、付款路径、拖延。
- 工作事业：资源、组织、节点、阻力。
- 出行迁移：驿马、阻隔、变动、归期。
- 感情关系：回应、和合、误会、暗处牵制。
- 健康忧患：轻重缓急、检查提示、不可替代医疗。

验收标准：
- 输出包含总象、所问分析、风险、行动建议、复盘观察点。
- 每条断语能解释为什么这么断。

## 第四阶段：案例复盘系统

让预测变成学习，而不是感觉。

- 案例云端保存。
- 支持后续补录结果。
- 应验程度：应验、部分应验、未应验、待定。
- 偏差原因：问事不清、取象错误、时机错误、执行偏差、外部突发。
- 按事项类型、神将组合、课象标签统计准确率。

验收标准：
- 能看到长期案例库。
- 能知道哪些类型准，哪些类型不准。

## 第五阶段：老师模式

系统不只给结论，还教人怎么看。

- 白话解释给普通用户。
- 术语解释给学习者。
- 点击断语可展开依据。
- 自动生成复盘提问。
- 将优秀案例沉淀成学习材料。

验收标准：
- 用户能从每次起课中学到取象方法。

## 第六阶段：服务器化与个人系统

把工具从静态页面升级为长期使用系统。

- 腾讯云服务器部署。
- 账号与个人案例库。
- 数据备份。
- 管理后台。
- 后续可接入安老师，作为解释和复盘教练。

验收标准：
- 个人长期复盘数据不丢。
- 公开用户可一次性体验，老王自己可长期学习。
